# Decisions Log

Calibration learnings, threshold tunes, and architectural shifts that need to survive context loss.

See CLAUDE.md §19 for the rules. Format:

- **Headline:** one line — what was decided / learned
- **Context:** what triggered it (commit hash, sub-step, symptom)
- **Resolution:** what changed (file refs, line refs)
- **Why it matters next time:** the part that survives context loss

Filename convention: `<utc-yyyy-mm-dd>__<topic-kebab>.md`

MEMORY.md keeps only one-line pointers to these files. Decisions log is authoritative.
