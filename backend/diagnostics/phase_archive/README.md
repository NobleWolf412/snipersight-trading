# Phase Archive

Frozen records of completed SniperSight build phases. Migrated out of MEMORY.md to keep the active memory index lean.

These are historical artifacts — what shipped, when, why. Not authoritative for current behavior (read the code for that), but useful when reconstructing what a commit landed and what the calibration round around it learned.

Filename convention: `<utc-yyyy-mm-dd>__<phase-or-topic-kebab>.md`

If you find yourself wanting to revive a memory entry from here, ask whether the underlying fact is still relevant. If yes, write a fresh decisions-log entry referencing this archive — don't unarchive.
